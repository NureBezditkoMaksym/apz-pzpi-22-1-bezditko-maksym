3 ОПИС ВИКОНАНОЇ РОБОТИ
Перенесемо попередню версію Vision & Scope документу.
	Документ стосується Програмної системи для контролю здорового харчування, яка забезпечує автоматизований моніторинг основних показників здоров’я користувачів (спожиті калорії, випита вода, рівень фізичної активності) шляхом інтеграції IoT-пристроїв для нагадувань, веб-інтерфейсу для аналітики та мобільного додатку для зручного щоденного внесення даних. Система дозволяє своєчасно нагадувати користувачам про необхідність обліку показників, автоматично формує аналітичні звіти та забезпечує візуалізацію динаміки, що сприяє формуванню корисних звичок та досягненню цілей у сфері здоров’я. Такий підхід підвищує мотивацію користувачів, забезпечує індивідуалізований підхід та сприяє профілактиці захворювань шляхом регулярного моніторингу стану здоров’я.
	Виправимо існуючі помилки. Опишемо новий функціонал, що стосується Web і Mobile складників застосунку. Додамо інформацію про те, як у майбутньому система може бути покращена завдяки ШІ — через персоналізовані рекомендації, адаптивну аналітику та інтелектуальні поради щодо харчування та фізичної активності на основі поведінкових патернів користувача.
	Фінальна версія документа включена в додаток Б.

Бізнес-вимоги
Бізнес-вимоги слугують основою та довідником для розробки всіх детальних вимог. Вони були зібрані від потенційних користувачів, фахівців у сфері здоров’я, консультантів зі спортивного способу життя та продукт-менеджерів — усіх, хто має чітке розуміння причин реалізації цього проєкту та його остаточної цінності як для користувачів, так і для індустрії здорового способу життя.

Передумови
Необхідність створення програмної системи для контролю здорового харчування виникла через зростаючий попит на інструменти, що допомагають людям систематизувати дані про свій стан здоров’я та формувати корисні звички. Існуючі рішення часто не забезпечують комплексного підходу, мають перевантажений інтерфейс або не інтегруються з пристроями IoT, що знижує регулярність обліку та мотивацію користувачів. Враховуючи важливість щоденного контролю спожитих калорій, води та фізичної активності для профілактики захворювань та підтримання гарного самопочуття, виникла потреба в сучасному рішенні. Наша система спрямована на впровадження інтуїтивного інтерфейсу, мобільного та веб-клієнтів, автоматизованих нагадувань через IoT-пристрої та можливість подальшого використання ШІ для персоналізованих порад.
Бізнес-можливості
У сучасному суспільстві, де здоровий спосіб життя є ключовим трендом, люди все частіше шукають ефективні засоби контролю свого харчування та фізичної активності. Проте більшість поточних рішень або мають вузьку спеціалізацію, або не забезпечують зручності у щоденному використанні. Додатковим бар’єром є відсутність персоналізації та гейміфікованої системи мотивації.
Наш продукт заповнює цю нішу, пропонуючи платформу, яка поєднує внесення даних про калорії, воду та активність із автоматизованими сповіщеннями, веб-інтерфейсом для аналітики та мобільним застосунком для щоденного користування. У майбутньому планується використання ШІ для генерації персоналізованих рекомендацій, прогнозів та мотиваційних порад. Це дозволить користувачам підвищити усвідомленість щодо власного стану здоров’я, покращити щоденну дисципліну та досягати своїх фітнес-цілей.
На ринку спостерігається зростаючий попит на універсальні, персоналізовані рішення для контролю здоров’я, особливо після пандемії та з поширенням wearable-пристроїв. Наша система повністю відповідає цим запитам, впроваджуючи сучасні цифрові технології та принципи повсякденного зручного моніторингу.
Бізнес-цілі та критерії успіху
Наші ключові бізнес-цілі включають:
Збільшення регулярності обліку показників здоров’я користувачами на 50% протягом першого року завдяки впровадженню нагадувань та зручного інтерфейсу.


Підвищення рівня залученості користувачів на 35% за рахунок мобільного додатку та системи гейміфікованих досягнень.


Досягнення конверсії у преміум-підписку на рівні 20% серед активних користувачів протягом першого року завдяки впровадженню персоналізованої аналітики.


Розширення аудиторії до 50 000 користувачів протягом двох років через маркетинг у соціальних мережах та співпрацю з фахівцями зі здоров’я.


Критерії успіху проєкту:
Кількісні показники: Аналіз частоти внесення даних, утримання користувачів, обсягів преміум-підписок та загальної активності в системі.


Дотримання графіків: Завершення ключових етапів розробки згідно з планом для своєчасного запуску на ринок.


Задоволеність користувачів: Позитивні оцінки в маркетплейсах, відгуки у соцмережах, низький рівень відмов від сервісу.


Відповідність стандартам: Дотримання стандартів безпеки даних (GDPR, HIPAA), а також вимог щодо обробки персональної та медичної інформації.


Для досягнення цих цілей ми зосередимося на якісній розробці продукту, мобільності, UX-дослідженнях, використанні AI-моделей для персоналізації та постійній підтримці користувачів. Враховуючи поточні тенденції та технічні ресурси, існують усі передумови для досягнення поставлених бізнес-результатів.


Потреби клієнтів та ринку
Люди, які прагнуть вести здоровий спосіб життя, стикаються з низкою викликів у систематичному контролі харчування та фізичної активності. Основні потреби користувачів та ринку включають:
Точний облік показників: Необхідність у простому способі внесення та перегляду даних про калорії, воду, кроки та інші метрики.


Регулярні нагадування: Система має стимулювати регулярність внесення даних через сповіщення, що надходять у мобільному застосунку або через IoT-пристрої.


Візуалізація прогресу: Користувачі потребують графіків та аналітики, що дозволяє оцінити ефективність змін у способі життя.


Персоналізовані поради: Очікується впровадження ШІ для формування рекомендацій на основі особистих даних і звичок.


Інтеграція з іншими сервісами: Важлива можливість підключення до фітнес-браслетів, розумних ваг, сторонніх трекерів калорій тощо.


Конфіденційність: Гарантії захисту персональних даних, зокрема медичної інформації.


Існуючі рішення часто не задовольняють усі ці потреби одночасно, особливо щодо персоналізації та простоти використання. Наша система надає комплексний, сучасний інструмент для моніторингу стану здоров’я, який поєднує гнучкість, зручність і точність на одній платформі.


Бізнес-ризики
Конкуренція на ринку: На ринку вже існують компанії, які пропонують схожі рішення для контролю харчування та фізичної активності. Наприклад, додатки “MyFitnessPal”, “YAZIO” та “Lifesum” надають широкий функціонал з підрахунку калорій, трекінгу води й активності. Їхні продукти мають міцну позицію на ринку та користуються довірою мільйонів користувачів.


Серйозність: Середня.


Стратегія пом’якшення:


Відмінні риси продукту: Розробити унікальні функції, які відрізняють наш продукт від конкурентів, такі як гейміфіковані досягнення, інтеграція з IoT-пристроями для нагадувань, персоналізовані поради на основі ШІ, локалізовані бази продуктів.


Партнерства та співпраця: Встановити стратегічні партнерства з фітнес-клубами, освітніми платформами та дієтологами для розширення впливу на аудиторію.


Якісна підтримка клієнтів: Забезпечити високий рівень обслуговування та підтримки, що підвищить задоволеність користувачів та сприятиме їх утриманню.


Маркетинг та брендинг: Провести активну маркетингову кампанію, яка підкреслить переваги нашого рішення та підвищить впізнаваність бренду на ринку.


Низька прийнятність користувачами: Користувачі можуть неохоче переходити на нову систему через звичку до старих методів або втому від цифрових рішень.


Серйозність: Висока.


Стратегія пом’якшення: Проведення навчальних кампаній, надання демонстраційних версій, гнучке онбординг-введення в систему, підтримка користувачів на всіх етапах використання.


Технічні складнощі: Можливі проблеми з інтеграцією системи з наявними смарт-пристроями (фітнес-браслети, ваги) або збої у синхронізації даних.


Серйозність: Середня.


Стратегія пом’якшення: Забезпечення сумісності через відкриті API, гнучкі налаштування та підтримка різних платформ.


Регуляторні зміни: Зміни в законодавстві або галузевих стандартах щодо обробки персональних або медичних даних можуть вимагати додаткових ресурсів для адаптації системи.


Серйозність: Низька.


Стратегія пом’якшення: Постійний моніторинг законодавчих змін, гнучка архітектура системи для швидкого внесення необхідних змін.


Фінансові ризики: Перевищення бюджету на розробку або невідповідність прогнозованим фінансовим показникам.


Серйозність: Середня.


Стратегія пом’якшення: Ретельне планування бюджету, регулярний фінансовий контроль, гнучке масштабування команди, можливість залучення зовнішнього фінансування у разі потреби.


Загрози безпеці даних: Можливість витоку конфіденційної інформації або несанкціонованого доступу до персональних даних користувачів.

Бачення рішення
Цей розділ встановлює довгострокове бачення системи для досягнення бізнес-цілей. Бачення надає контекст для прийняття рішень протягом усього життєвого циклу розробки продукту. Воно не повинно включати детальні функціональні вимоги або інформацію про планування проєкту.
Заява про бачення
Ми прагнемо створити просту, але надійну програмну систему для контролю показників здоров’я, що дозволить ефективно відстежувати спожиті калорії, випиту воду та рівень фізичної активності, з можливістю отримання нагадувань через IoT-пристрої.
Розширення функціональності включає впровадження персоналізованої аналітики за допомогою ШІ, формування рекомендацій щодо харчування та активності, а також захищену систему збереження та обміну даними між клієнтами й сервером.
Мобільна версія буде реалізована на основі Swift із забезпеченням основного функціоналу CRUD для користувачів та історії показників.
Інтерфейс веб-клієнта, розроблений з використанням React, передбачає реалізацію операцій CRUD для користувачів, перегляду аналітики та управління профілем. Функціонал звітності та імпорту/експорту даних буде доступним для користувачів із преміум-доступом із захисним паролем.
В рамках демонстрації можливостей проєкту планується автоматизація розгортання за допомогою скриптів, які інтегрують серверну частину, базу даних, веб-інтерфейс та мобільний застосунок у єдину систему, що сприятиме швидкому впровадженню та тестуванню продукту.
Основні функції
Система забезпечує широкі можливості для управління персональними показниками здоров’я. Вона дозволяє вести щоденний облік калорій, води та фізичної активності користувача, з можливістю додавання фотозаписів для візуального контролю прогресу.
Завдяки інтеграції з IoT-пристроями, система надсилатиме нагадування про необхідність внесення показників, що сприяє формуванню корисних звичок.
Автоматично генеруються аналітичні звіти про динаміку стану здоров’я, включаючи графіки, середні значення та індивідуальні рекомендації на основі отриманих даних.
Система підтримує функціональність синхронізації між пристроями, а також дозволяє імпорт і експорт даних користувача у захищеному форматі.
Для користувачів преміум-рівня доступна розширена статистика, аналітика з використанням штучного інтелекту, а також підтримка створення цілей (наприклад, «випивати 2 л води щодня» або «схуднути на 5 кг за 2 місяці»).
Припущення та залежності
Наш проєкт базується на кількох важливих припущеннях, які визначають його подальшу реалізацію.
	По-перше, ми виходимо з того, що користувачі мають базові навички роботи зі смартфоном або ПК, що дозволяє їм ефективно користуватися системою без додаткового навчання.
	По-друге, передбачається, що користувачі мають доступ до інтернету для синхронізації даних та отримання сповіщень.
	Крім того, очікується, що частина користувачів буде готова встановити IoT-пристрій (на базі Arduino або інші) для отримання автоматичних нагадувань.
Залежності проєкту включають:
використання надійних IoT-пристроїв для генерації нагадувань;


стабільність серверної інфраструктури та наявність інтеграції з системами автентифікації та платіжними платформами (наприклад, Stripe);


дотримання вимог щодо обробки персональних та чутливих даних відповідно до стандартів GDPR/ISO.

Детальний опис функцій системи та вимоги
Опис початкової системи
Цей розділ визначає початковий обсяг функціональності системи, яка забезпечує комплексне рішення для контролю здорового способу життя. Система надає можливості для щоденного обліку харчування та активності, автоматичного нагадування про внесення показників, аналітики прогресу, а також інтегрує серверну обробку даних, IoT-нагадування, веб- та мобільний інтерфейси й інструменти для захищеного управління даними та автоматичного розгортання.

3.1.1. Функціональність серверної частини:
Управління даними: зберігання та обробка інформації про користувачів, щоденні показники (калорії, вода, активність), фотозаписи, цілі та підписки.
Обробка запитів: прийом і швидка обробка запитів від веб- та мобільних клієнтів.
Взаємодія з IoT-пристроями: надсилання команд для нагадувань користувачу у визначений час.
Автоматичні сповіщення: формування нагадувань про необхідність внесення щоденних даних.
Генерація звітів: формування звітів про динаміку показників здоров’я з візуалізацією.
Безпека та автентифікація: забезпечення захищеного доступу та прав користувачів.
Експорт/імпорт даних: захищене збереження, експорт і імпорт даних користувача із шифруванням.

3.1.2. Вимоги до серверної частини:
Надійність: безперебійна робота з мінімальними простоями.
Масштабованість: підтримка великої кількості користувачів та даних.
Безпека: відповідність GDPR, використання шифрування.
Сумісність: можливість інтеграції з API носимих пристроїв (наприклад, Fitbit).
Продуктивність: висока швидкість обробки запитів.

3.1.3. Функціональність IoT-клієнта:
Генерація нагадувань: звукові або світлові сигнали про необхідність внести дані.
Взаємодія з сервером: прийом команд на нагадування.
Моніторинг стану: повідомлення про технічні несправності пристрою.

3.1.4. Вимоги до IoT-клієнта:
Точність: чітке виконання нагадувань у визначений час.
Надійність зв'язку: стабільна синхронізація з сервером.
Енергоефективність: мінімальне споживання енергії.
Простота встановлення: інструкція для користувача без технічних знань.
Захищеність: захист від фізичного втручання та збоїв.

3.1.5 Функціональність веб-клієнта:
Облік показників: додавання, редагування, перегляд даних про калорії, воду, активність.
Фотофіксація: завантаження фото для візуального трекінгу.
Звіти та аналітика: графіки, таблиці, динаміка змін.
Експорт/імпорт даних: збереження інформації з шифруванням.
Управління цільовими параметрами: створення та редагування персональних цілей.

3.1.6 Вимоги до веб-клієнта:
Інтерфейс: зручний та інтуїтивний.
Сумісність: підтримка усіх сучасних браузерів.
Безпека: авторизація, захист даних.
Відповідність: сучасні стандарти front-end розробки.
Продуктивність: швидка відповідь інтерфейсу.

3.1.7. Функціональність мобільного клієнта:
Доступ до даних: перегляд і внесення показників.
Сповіщення: отримання нагадувань.
Фотозаписи: додавання з галереї або камери.
Синхронізація: обмін даними з сервером.

3.1.8. Вимоги до мобільного клієнта:
Інтерфейс: адаптований до мобільних екранів.
Ефективність: мінімальні затрати ресурсів.
Безпека: автентифікація, шифрування даних.
Офлайн-режим: внесення даних без мережі з подальшою синхронізацією.

3.1.9. Розгортання та демонстрації:
Реалізація автоматичних скриптів для розгортання серверної частини, бази даних, веб-клієнта та мобільного застосунку як єдиної інтегрованої системи для тестування і демонстрації.

3.1.10. Опис всієї системи:
Система є комплексним рішенням для контролю здорового способу життя, поєднуючи серверну логіку, нагадування через IoT, зручні клієнтські інтерфейси, аналітику та захист даних. Розширена функціональність та автоматизоване розгортання забезпечують ефективну реалізацію та масштабованість продукту, надаючи користувачам повноцінний інструмент для турботи про власне здоров’я.
Опис майбутніх покращень системи
У майбутніх релізах системи планується інтеграція технологій штучного інтелекту (ШІ) для ще більшої персоналізації користувацького досвіду та підвищення ефективності контролю показників здоров’я. У цьому контексті ми розглядаємо три ключові аспекти роботи з ШІ

3.2.1. Застосування ШІ
ШІ дозволить значно покращити функціональність системи, забезпечуючи:
персоналізовані поради щодо харчування, споживання води та активності на основі аналізу індивідуальної історії користувача;
виявлення відхилень від нормального ритму життя (наприклад, раптове падіння фізичної активності);
рекомендації щодо досягнення поставлених цілей або їх коригування;
прогнозування майбутніх змін ваги або рівня гідратації за поточними звичками.
Також передбачається застосування ШІ для виявлення шаблонів у поведінці користувача, формування адаптивного плану дій, автоматичного створення щотижневих рекомендацій та мотивуючих повідомлень.
3.2.2. Збирання даних:
Для навчання моделей ШІ система буде збирати великі обсяги структурованих даних, зокрема:
щоденні показники користувача (калорії, вода, кроки, вагові зміни);
фотозаписи, історія досягнутих цілей;
активність у мобільному та веб-додатку (регулярність внесення, частота переглядів аналітики);
дані з IoT-пристроїв (наприклад, час взаємодії з нагадуваннями).
Зібрані дані будуть анонімізовані й агреговані для дотримання конфіденційності та відповідності стандартам захисту персональної інформації (GDPR). Це дозволить створити високоточні прогностичні моделі для рекомендацій, моніторингу ризиків та гейміфікованої підтримки користувачів.
3.2.3. Навчання та використання моделей ШІ:
У перспективі передбачається кілька напрямів використання моделей ШІ:
Власна розробка: створення алгоритмів, що базуються на накопичених даних користувачів для персонального прогнозування.
Донавчання сторонніх моделей: використання існуючих моделей (наприклад, OpenAI, Hugging Face), адаптованих під специфіку поведінки користувачів системи.
Інтеграція зовнішніх API: застосування готових сервісів для швидкого впровадження функцій ШІ без значних витрат.


Такий гнучкий підхід дозволить постійно вдосконалювати систему, підвищуючи рівень персоналізації, мотивації користувача та ефективності досягнення здорового способу життя.
Обмеження системи
Попри те, що початковий реліз системи охоплює більшість ключових функцій для ефективного контролю показників здоров’я, існують деякі обмеження та функціональність, яка могла б бути присутньою, але свідомо виключена.
	По-перше, система на даному етапі не включає розширених засобів візуалізації — таких як інтерактивні графіки, гнучке сортування та фільтрація даних у дашбордах. Це зроблено свідомо для зменшення складності інтерфейсу, зниження порогу входу для користувача та прискорення реалізації MVP.
	По-друге, хоча застосування ШІ є одним з основних напрямів розвитку, початковий реліз не включає автоматизованих рекомендацій, індивідуального прогнозування або виявлення шаблонів поведінки. На старті акцент зроблено на збиранні достовірних структурованих даних та стабільній роботі основного функціоналу.
	Крім того, система наразі реалізує лише базову інтеграцію з IoT-пристроями через стандартні інтерфейси, без глибокої адаптації під різні моделі або сторонні екосистеми (як-от Fitbit або Apple HealthKit).
	Також у початковому варіанті відсутні розширені механізми валідації користувацьких даних при введенні (наприклад, перевірка на конфлікт метрик або нереалістичні значення), а система профілювання користувачів реалізована мінімально — без багаторівневих ролей чи кастомізації інтерфейсу.
	Таким чином, ці обмеження спрямовані на спрощення перших етапів розгортання, стабільність роботи та поступову реалізацію вдосконаленого функціоналу в наступних релізах.

Зацікавлені сторони, строки виконання та використовувані технології
Зацікавлені сторони
Кінцеві користувачі: Особи, які бажають контролювати свої харчові звички, водний баланс і рівень фізичної активності.
Преміум-користувачі: Користувачі, що потребують розширеної аналітики, персоналізованих порад та цілей.
Фітнес-тренери, дієтологи: Спеціалісти, що використовують аналітику користувача для супроводу клієнтів.
Розробницька команда: Відповідальна за розробку, підтримку та вдосконалення системи.
Партнерські сервіси: Постачальники IoT-пристроїв, фітнес-браслетів або платіжних API (Stripe).
Маркетинг та аналітика: Внутрішні команди, що використовують агреговані дані для формування стратегії розвитку.
Строки виконання проєкту
Проєкт планується завершити до середини 2025 року.
Використовувані технології
React: JavaScript-бібліотека для розробки інтерфейсу веб-клієнта з підтримкою інтерактивних компонентів.
Swift: Мова програмування для розробки нативного мобільного додатку під iOS. Забезпечує високу продуктивність, плавність роботи інтерфейсу та глибоку інтеграцію з системними можливостями iPhone.
Express (Node.js): Серверний фреймворк для обробки API-запитів, логіки автентифікації та зв’язку з базою даних.
Supabase: Платформа, що поєднує зберігання даних, автентифікацію, Edge-функції та бекенд-інфраструктуру для швидкого розгортання.
Arduino: Використовується для реалізації IoT-нагадувань (світло/звук) через зовнішні пристрої, синхронізовані з сервером.
Stripe API: Інструмент для реалізації платіжних функцій, зокрема оформлення та керування преміум-підпискою.
OpenAI API (у майбутньому): Для реалізації функцій персоналізованих рекомендацій, аналізу даних і роботи зі штучним інтелектом.

Ці технології обрані для забезпечення надійності, продуктивності та зручності використання системи, що відповідає потребам користувачів та вимогам проєкту.
